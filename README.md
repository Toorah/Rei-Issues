# Rei — Issues & Feedback

Public issue tracker and feedback hub for **Rei**, a fast Windows desktop **Unreal Engine launcher** with a built-in **MCP bridge** — an LLM ⇄ Rei ⇄ Unreal control plane that lets an agent manage engine processes, edit settings, configure plugins, and drive the editor.

The Rei application itself is closed-source. **This repository has no code** — it exists purely for **bug reports, feature requests, and discussion**. File anything here and we'll triage it.

## File a report

- 🐛 **Bug** — [open an issue](../../issues/new/choose)
- 💡 **Feature request / idea** — [open an issue](../../issues/new/choose)
- 💬 **Questions & discussion** — [Discussions](../../discussions)

Please **search existing issues** first to avoid duplicates — 👍 an existing one rather than filing again.

## What to include

A good bug report has:

- **Rei version** — Settings → About
- **Windows version** (e.g. Windows 11 23H2)
- **Unreal Engine version** (e.g. 5.7.4)
- **Steps to reproduce**, what you expected, and what happened
- For **MCP bridge** issues: which client (Claude Desktop / Claude Code / Cursor / VS Code), the transport, and any relevant lines from the bridge log

## Security & private reports

Please **do not** file security-sensitive reports as public issues. Reach us privately through the **contact form on the website** — https://rei-tools.app — and we'll handle it there. *(A dedicated contact/security form is coming; this section will link it directly once live.)*
